​**一、环境搭建​

SQLiLabs 需要本地运行，推荐用 ​Windows + PHP环境​（新手友好）。以下是详细步骤：

​1. 下载 SQLiLabs​

  • 访问 GitHub 搜索 sqli-labs（或直接访问 https://github.com/Audi-1/sqli-labs），点击 Code → Download ZIP 下载压缩包。
  • 解压到电脑任意位置（比如 D:\sqli-labs）。

​2. 安装 PHP 环境​

SQLiLabs 依赖 PHP + MySQL 运行，推荐用 ​phpStudy​（集成环境，一键安装）：

  • 搜索并下载 phpStudy 2023（或旧版，兼容即可），安装时一路默认。
  • 打开 phpStudy，启动 Apache 和 MySQL（按钮变绿即成功）。

​3. 配置 SQLiLabs​

  • 打开 D:\sqli-labs 文件夹，找到 sql-connections 文件夹，双击打开 db-creds.inc 文件（用记事本）。

  • 修改数据库连接信息（默认可能已配置，若报错再改）：

  • 保存文件，回到 SQLiLabs 根目录，双击 setup.php（会自动创建数据库和表）。

  • 若提示“数据库连接失败”，检查：

    • MySQL服务是否启动（phpStudy里确认）；
    • db-creds.inc 中的用户名密码是否和 MySQL一致（phpStudy默认root/空）。

​三、第二关：布尔盲注（Less-2）​​

​目标​：页面不报错，但会根据注入条件的真假返回不同内容（比如显示“You are in”或空白），通过这种差异推断数据。

​1. 判断注入类型​

访问 http://localhost/sqli-labs/Less-2/?id=1，页面正常显示。
输入 id=1'，页面可能显示空白或提示“非法参数”（没有报错），说明是布尔盲注​（依赖页面返回的真假判断）。

​2. 布尔盲注的核心逻辑​

通过构造 1' and 条件 --+ 的语句，让数据库判断条件是否成立：

  • 如果条件为真（1' and 1=1 --+），页面正常；
  • 如果条件为假（1' and 1=2 --+），页面异常（比如空白）。

我们需要知道数据库名的长度（比如 security 是8位）。构造payload：

  • 若页面正常，说明长度等于8；若异常，说明长度不是8。
  • 可以用二分法试探（比如 length(database())>5 → 正常；length(database())>7 → 异常 → 长度是8）。

​4. 逐字符获取数据库名​

用 ASCII() 和 SUBSTRING() 函数逐个字符猜解。例如，猜第一个字符：

如果页面正常，说明第一个字符是 s；否则，换其他ASCII码（比如 101 对应 e）。

​四、第三关：时间盲注（Less-3）​​

​目标​：页面既不报错也不返回真假，但会根据条件延迟响应（比如条件成立时延迟5秒），通过时间差判断数据。

​1. 判断注入类型​

访问 http://localhost/sqli-labs/Less-3/?id=1，页面正常。
输入 id=1'，页面可能无变化（既不报错也不空白），但实际存在注入。

​2. 时间盲注的核心逻辑​

利用 SLEEP(N) 函数让数据库延迟 N 秒，通过页面响应时间判断条件是否成立：

  • 如果 1' and sleep(5) --+ 页面5秒后加载，说明条件为真；
  • 如果立即加载，说明条件为假。
